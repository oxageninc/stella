Give Stella a goal and let it work in **judged rounds** until a judge model confirms, from evidence, that the goal has actually been met.

## Synopsis

```bash
stella goal <goal> [global flags]
```

## What it does

`stella goal` runs the agent step loop in rounds. At the end of each round, a **judge model** assesses whether the goal is met based on the evidence produced that round. The loop ends only when the judge confirms success.

Crucially, the judge is routed **cross-family** — it is a different model family than the worker, to avoid same-model bias. If only one provider family is configured, the worker doubles as its own judge.

<Callout type="info">
For the full explanation of judged rounds, the cross-family judge, and the definition-of-done check, see [Goal mode](/docs/goal-mode).
</Callout>

## Flags

`stella goal` takes the [global flags](/docs/commands). The most relevant ones:

| Flag | Why it matters here |
| --- | --- |
| `--model <provider/model_id>` | Pin the worker model. The judge is routed to a different family automatically when one is available. |
| `--budget <usd>` | Cap total spend across all rounds; work aborts cleanly once the cap is exceeded. |
| `--output-format <text\|json\|stream-json>` | Use `json` or `stream-json` for headless, machine-readable output. |

<Callout type="warn">
Configure at least two provider families if you want a truly independent judge. With a single family configured, the worker model judges its own work.
</Callout>

## Examples

Work toward a goal until the judge confirms it is met:

```bash
stella goal "Migrate the config loader to use serde and add tests"
```

Pin the worker model and cap spend across all rounds:

```bash
stella goal "Make the CLI exit with a non-zero code on invalid input" \
  --model anthropic/claude-fable-5 \
  --budget 10.00
```

Run headless and stream each round's events:

```bash
stella goal "Get the test suite to 100% pass rate" \
  --output-format stream-json
```

You can also enter goal mode from an interactive session with the [`/goal`](/docs/commands/chat) slash command.
