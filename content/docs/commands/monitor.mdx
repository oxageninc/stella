Watch CI for a branch or pull request and fix failures until the pipeline is fully green.

## Synopsis

```bash
stella monitor [target] [global flags]
```

The optional `target` is a branch or PR to watch. When omitted, it defaults to `main`.

## What it does

`stella monitor` uses the `ci_status` tool to watch the CI runs for the given `target`. It works iteratively — inspecting failures and applying fixes — and treats the job as a goal that is met **only when the latest CI run is all-green**.

Because it is goal-shaped, `monitor` keeps working across CI runs until the pipeline passes, then exits.

<Callout type="info">
`stella monitor` relies on the `ci` tool for CI status and logs. See [Tools](/docs/commands/tools) for the full built-in tool list.
</Callout>

## Flags

`stella monitor` takes the [global flags](/docs/commands). The most relevant ones:

| Flag | Why it matters here |
| --- | --- |
| `--model <provider/model_id>` | Pin the worker model used to diagnose and fix failures. |
| `--budget <usd>` | Cap total spend while it works across CI runs. |
| `--output-format <text\|json\|stream-json>` | Use `json` or `stream-json` for headless CI output. |

## Examples

Watch the default `main` branch and fix failures until green:

```bash
stella monitor
```

Watch a specific branch:

```bash
stella monitor feature/payments-refactor
```

Watch a pull request and cap total spend:

```bash
stella monitor "#128" --budget 5.00
```
