title: Permissions

Every tool the agent can call — built-in, [custom](/docs/tools/custom-tools), or provided by an [MCP server](/docs/tools/mcp) — has a per-tool permission model. This is what lets Stella treat a harmless file read differently from a shell command that can change your system.

## Read-only vs. mutating tools

Tools fall into two categories:

- **Read-only tools** only observe your workspace and environment. They never change files, run commands, or perform side effects. These tools are marked `read_only: true` in their permission model. Examples from the [built-in catalog](/docs/tools) include `read`, `glob`, `grep`, `ci`, `screenshot`, `verify`, `exploration`, and `registry`.
- **Mutating and shell tools** can change files, execute commands, or perform other side effects. Examples include `write`, `edit`, `delete`, `bash`, `exec`, `project`, `memory`, and `issues`.

The `read_only: true` marker is the signal that a tool is safe to run without changing anything. Tools without it should be treated as capable of side effects.

<Callout type="info">
Read-only vs. mutating is a property of the tool, not of a specific invocation. If you want fine-grained, per-invocation control — for example, blocking `bash` when a command matches a dangerous pattern — use a hook, described below.
</Callout>

## Gating tool calls with hooks

The permission model is enforced ahead of each tool call, and it can be extended with lifecycle hooks. In particular, a **`PreToolUse` hook** runs before a tool executes and can **block** it: if the hook exits with a non-zero status, the tool does not run and the model receives the hook's message instead.

This gives you a programmable gate in front of any tool. A `PreToolUse` hook's matcher is a glob over the tool name, so you can target a single tool (for example, `bash`) or a family of tools.

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "bash",
        "hooks": [
          {
            "type": "command",
            "command": "./scripts/guard.sh",
            "timeoutMs": 5000
          }
        ]
      }
    ]
  }
}
```

For the full hook model — events, matchers, timeouts, and the JSON payload delivered on stdin — see [Hooks](/docs/tools/hooks).
