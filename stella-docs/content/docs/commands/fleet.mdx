---
title: stella fleet
description: Fan a batch of tasks out to a fleet of worker agents, each isolated in its own git worktree.
---

Run many tasks in parallel, each in its own isolated git worktree, wave-scheduled by dependency. Every attempt, commit, and dollar is recorded in `.stella/fleet.db`, and the worktrees and their `fleet/<task>` branches are left in place for review.

## Synopsis

```bash
stella fleet <task>... [--plan <FILE>] [--max-concurrency <N>] [--base-ref <ref>] [--watch]
```

## What it does

Each task prompt becomes an independent, isolated task. Stella creates one git worktree per task — branched from the current `HEAD` (or `--base-ref`) onto a `fleet/<task>` branch — and dispatches tasks in dependency-ordered waves, running up to `--max-concurrency` at once within a wave. Progress, attempts, commits, and cost are persisted to `.stella/fleet.db`.

Worktrees and branches are **not** cleaned up automatically: they stay on disk so you can inspect, test, and merge each result yourself.

<Callout type="info">
For anything beyond a handful of independent prompts, use a plan file (`--plan`). It lets you declare task `id`, `title`, `prompt`, `depends_on` (to order the waves), and `claims` — file paths held as cooperative locks while a task runs, so two tasks never edit the same file at once.
</Callout>

## Flags

| Flag | Meaning |
| --- | --- |
| `<task>...` | One or more task prompts, each an independent isolated task. Required unless `--plan` is given. |
| `--plan <FILE>` | A `.json` or `.toml` plan file with `[[tasks]]` entries (`id`, `title`, `prompt`, optional `depends_on`, `isolation`, and `claims`) instead of inline prompts. |
| `--max-concurrency <N>` | Maximum tasks dispatched concurrently within one wave. Default `4`. |
| `--base-ref <ref>` | Git ref the isolated worktrees branch from. Default: the current `HEAD`. |
| `--watch` | After the fan-out, watch each `fleet/<task>` branch's CI to completion and reconcile its PR status via `gh`. Exits non-zero if any watched branch ends red. |

## Examples

Fan three independent tasks out across worktrees:

```bash
stella fleet \
  "Add unit tests for the parser module" \
  "Document every public function in src/api/" \
  "Fix the clippy warnings in the store crate"
```

Run a dependency-ordered plan and cap concurrency at two:

```bash
stella fleet --plan .stella/plans/release-prep.toml --max-concurrency 2
```

Branch from a release branch and watch each branch's CI after the run:

```bash
stella fleet --plan release.toml --base-ref release/1.4 --watch
```

<Callout type="warn">
`--watch` (and its PR reconciliation) only does something once the fleet branches are pushed — for example, when your task prompts push their branch and open a PR. It uses the GitHub `gh` CLI, which must be installed and authenticated.
</Callout>
